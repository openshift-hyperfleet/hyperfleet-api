## Commands

### Setup (fresh clone)
```
make generate-all     # REQUIRED FIRST ŌĆö generated code not in git
go mod download
make install-hooks    # Install pre-commit hooks (secret scanning, linting, etc.)
make db/setup         # Start local PostgreSQL container
make build            # Build binary (CGO_ENABLED=1 GOEXPERIMENT=boringcrypto)
./bin/hyperfleet-api migrate
make run-no-auth      # Start server without auth
```

### Build & Run
```
make build            # Build hyperfleet-api binary to bin/
make install          # Build and install to GOPATH/bin
make run              # Build, migrate, and run with auth
make run-no-auth      # Build, migrate, and run without auth
```

### Code Generation
```
make generate         # Extract schema from hyperfleet-api-spec module, then run oapi-codegen
make generate-mocks   # Regenerate mock implementations (go generate)
make generate-all     # Both of the above
```


## Testing

**Unit tests**: `make test` ŌĆö sets `HYPERFLEET_ENV=unit_testing`, runs `./pkg/...` and `./cmd/...`

**Integration tests**: `make test-integration` ŌĆö sets `HYPERFLEET_ENV=integration_testing` and `TESTCONTAINERS_RYUK_DISABLED=true`. Testcontainers auto-creates isolated PostgreSQL instances. Located in `test/integration/`.

**Helm tests**: `make test-helm` ŌĆö lints and renders templates with multiple value combinations.

**Mock generation**: `make generate-mocks` ŌĆö uses `go generate` directives with `go.uber.org/mock/gomock`. Never write mocks manually.

**Test factories**: `test/factories/` ŌĆö create resources via the service layer, not directly in DB. Use `NewCluster()`, `NewClusterWithStatus()`, `NewClusterWithLabels()`.

**Integration test setup**: `test.RegisterIntegration(t)` returns `(helper, client)`. Uses Gomega assertions and Resty HTTP client.

**Environment variables for tests**:
- `HYPERFLEET_ENV` ŌĆö selects config: `unit_testing`, `integration_testing`, `development`
- `TESTCONTAINERS_RYUK_DISABLED=true` ŌĆö required in CI
- `HYPERFLEET_CLUSTER_ADAPTERS` / `HYPERFLEET_NODEPOOL_ADAPTERS` ŌĆö adapter lists (defaults set in TestMain)

## Code Style

### Imports
Order: stdlib ŌåÆ external ŌåÆ internal (`github.com/openshift-hyperfleet/hyperfleet-api/...`)

### Errors
Use constructor functions from `pkg/errors/errors.go`: `NotFound()`, `Validation()`, `GeneralError()`, `Conflict()`, `ValidationWithDetails()`. Error codes: `HYPERFLEET-CAT-NUM` format. All service methods return `*errors.ServiceError`.

### Logging
Use `pkg/logger/` ŌĆö `logger.Info(ctx, "msg")`, `logger.With(ctx, "key", val).Error("msg")`. Never use `fmt.Println` or `log.Print`.

### Handlers
Use `handlerConfig` pipeline from `pkg/handlers/framework.go`:
- `handle(w, r, cfg, status)` ŌĆö unmarshal ŌåÆ validate ŌåÆ action ŌåÆ respond
- `handleGet/handleList/handleDelete` ŌĆö no-body variants
- Validation: `func() *errors.ServiceError`; Action: `func() (interface{}, *errors.ServiceError)`

### Services
Interface + `sql*Service` struct. Constructor injection of DAOs. Return `*errors.ServiceError`. Add `//go:generate mockgen` directive for mocks.

### DAOs
Interface + `sql*Dao` struct. Get session via `sessionFactory.New(ctx)`. Call `db.MarkForRollback(ctx, err)` on write errors. Return stdlib `error`.

### Plugins
Register via `init()`: `registry.RegisterService()`, `server.RegisterRoutes()`, `presenters.RegisterPath()`, `presenters.RegisterKind()`. See `plugins/clusters/plugin.go`.

## Git Workflow

### Commit Format
```
HYPERFLEET-### - type: description
```
Types: `feat`, `fix`, `refactor`, `test`, `docs`, `chore`

Add co-author line for AI-assisted commits:
```
Co-Authored-By: Claude <noreply@anthropic.com>
```

### Pre-commit Hooks
Install: `make install-hooks`

Hooks:
- `leaktk.git.pre-commit` ŌĆö secret scanning (open-source, no VPN required)
- `hyperfleet-commitlint` ŌĆö validates commit message format (commit-msg stage)
- `hyperfleet-gofmt` ŌĆö Go code formatting
- `hyperfleet-golangci-lint` ŌĆö linting
- `hyperfleet-go-vet` ŌĆö Go vet checks
- `trailing-whitespace` ŌĆö removes trailing whitespace
- `end-of-file-fixer` ŌĆö ensures files end with newline
- `check-added-large-files` ŌĆö prevents large files from being committed

### Branching
Create feature branches from `main`. PRs target `main`.

## Boundaries

- **Never edit** `pkg/api/openapi/` or `*_mock.go` ŌĆö regenerate with `make generate-all`
- **Never set** `status.phase` manually ŌĆö calculated from adapter conditions
- **Never create** direct DB connections ŌĆö use `SessionFactory.New(ctx)` for transaction participation
- **FIPS required**: build with `CGO_ENABLED=1 GOEXPERIMENT=boringcrypto`
- **Spec source of truth**: `hyperfleet-api-spec` Go module; update `go.mod` to change spec versions ŌĆö see [openapi/README.md](openapi/README.md) for full details on schema import, code generation, and validation
- **Tool versions** managed by Bingo (`.bingo/`) ŌĆö don't manually install oapi-codegen or golangci-lint
